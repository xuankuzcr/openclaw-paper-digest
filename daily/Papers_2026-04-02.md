## 🧾 今日结论（先看这5条）
- **“世界模型”正在从机器人扩展到更广义的可计算世界。** 从视频生成、折纸规划到细胞转录组，大家都在用“可生成+可预测”框架做决策前推演，趋势是方法论统一、应用域分化。  
- **空地一体仿真基础设施开始补齐数据与评测短板。** CARLA-Air 这类平台把无人机与地面智能体放进同一环境，有望提升多体协同与跨视角感知研究效率，但仿真到真实（Sim2Real）偏差仍是落地主风险。  
- **方法走向上，生成模型正从“会造画面”转向“能做规划”。** Video world model 与 Learn2Fold 都体现了“生成-预测-搜索”闭环，说明扩散/视频模型正在进入任务级决策层，而非仅做感知前端。  
- **落地可行性取决于可控性与成本，而非模型规模本身。** 这些工作若要进工业/医疗场景，必须解决长时序稳定性、物理一致性与可解释评估；否则“看起来合理”的生成结果可能在执行时失效。  
- **机会在跨学科高价值场景，风险在错误世界模型引发系统性误导。** 虚拟细胞与具身智能共用世界模型范式，意味着机器人可借鉴生物建模的先验与验证机制；但一旦模型内隐偏差未被识别，规划层会放大错误并带来安全隐患。

### 🧠 World Models（3）

#### ⭐ Video Generation Models as World Models: Efficient Paradigms, Architectures and Algorithms
- **主题**：World Models, 4DGS + Diffusion Simulator
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.28489.png
![Video Generation Models as World Models: Efficient Paradigms, Architectures and Algorithms](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.28489.png)
- **这篇在做什么**：文章核心观点是：视频生成模型已经能学到物理与因果，但若算力开销过大，就难成为可用的世界模拟器。为此作者系统回顾了“如何更省算力地建模时空动态”，并给出统一分类框架。它更像一张路线图，帮助研究者在效果、速度、成本之间做工程化权衡。
- **核心方法**：作为综述，其“方法”是统一组织现有技术：输入/表示层面涵盖像素、潜空间与显式3D/4D表示；关键模块层面比较扩散、Transformer、时空解耦与稀疏计算等结构；流程层面梳理训练阶段的高效目标与数据利用，以及推理阶段的加速采样、缓存复用和长时生成策略。最终形成面向世界模型的效率优先技术栈。
- **和以前有什么不同**：相较以往只按生成范式或只看视觉质量的综述，这篇把“效率”提升为一等公民，并系统连接建模、架构与推理三层。它的增量不在提出单一新模型，而在提供跨层对照框架，解释为何很多方法在离线评测有效却难以实时交互部署。该视角更贴近真实应用闭环。
- **你能学到什么**：阅读时可先按你的约束倒推：若受限于时延，优先看推理算法；若受限于显存与训练成本，优先看表示与架构。把文中三维分类映射到自己的任务清单，能快速筛掉不合适路线。另一个收获是评估标准应加入吞吐、延迟与稳定性，而不只看感知质量。
- **关键洞见**：最重要的洞见是：世界模型能力与实用价值之间，真正的分水岭是效率而非单点画质。只有把高效表示、可扩展网络和低成本推理协同设计，视频生成才可能支持实时闭环决策。换言之，效率不是后处理优化，而是决定其是否成为通用模拟器的前置条件。

#### ⭐ Learn2Fold: Structured Origami Generation with World Model Planning
- **主题**：World Models, 4DGS + Diffusion Simulator
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.29585.png
![Learn2Fold: Structured Origami Generation with World Model Planning](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.29585.png)
- **这篇在做什么**：可以把它理解为“先让大模型出折法草案，再让物理世界模型提前验错”。传统方法要么太依赖精确几何输入、难接自然语言，要么能写得像却折不成。Learn2Fold把两者结合，在执行前就筛掉会碰撞或违背折纸约束的步骤，从而更稳定地产生可落地的折叠流程。
- **核心方法**：输入是抽象文本描述，内部表示为折痕图上的条件程序（折叠动作序列与结构约束）。关键模块包括：负责语义提案的LLM、预测可行性与失败模式的图结构世界模型，以及进行前瞻搜索的lookahead规划器。流程上先由LLM采样候选程序，再用可微代理模拟器打分与诊断，规划器迭代重排/修正候选，最终输出通过物理校验的长程折叠序列。
- **和以前有什么不同**：相较只做优化的方法，它不再要求稠密且精确的任务规格，而能从自然语言出发构造候选折叠程序。相较纯生成模型，它把“是否能折成”作为显式预测目标，并在规划环中持续验证，而非一次性生成。核心增量是语义提案与物理验证解耦，再通过世界模型闭环耦合。
- **你能学到什么**：可借鉴的工程要点是把世界模型当作“执行前审稿人”，先做失败模式预测，再投入真实执行或高代价仿真。对程序化机器人任务，可采用“候选生成—可行性评估—前瞻重规划”的通用模板。阅读时重点关注其折痕图状态定义、可微可行性信号设计，以及错误反馈如何反哺候选修正。
- **关键洞见**：最关键洞见是：在强约束长时程任务中，语言模型负责提出“可能对”的高层结构，世界模型负责判定“哪里会错”的物理细节。两者不是替代关系，而是通过规划形成互补闭环。真正的空间智能来自符号推理与具身物理预测的协同，而非单一大模型能力外推。

#### Lingshu-Cell: A generative cellular world model for transcriptome modeling toward virtual cells
- **主题**：World Models, 4DGS + Diffusion Simulator
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.25240.png
![Lingshu-Cell: A generative cellular world model for transcriptome modeling toward virtual cells](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.25240.png)
- **这篇在做什么**：可把Lingshu-Cell理解为一个会“补全与采样”细胞转录组的生成器：先学会真实细胞表达模式，再按条件生成可能出现的细胞状态。它不只给一个静态向量，而是学习整体现象的概率分布，所以能模拟异质性与亚群比例。加入细胞类型、供体和扰动条件后，还能预测干预后的全转录组变化。
- **核心方法**：输入是单细胞转录组在离散token空间中的表示，覆盖全基因范围而非预筛基因；条件信息可包含细胞类型、供体身份与扰动。核心模块是掩码式离散扩散生成框架，通过在token层面学习基因间高维依赖，适配稀疏且非序列的scRNA特性。训练时学习从被破坏/掩码状态恢复真实分布；推理时在给定条件下迭代去噪采样，生成整细胞表达谱并进行扰动响应预测。
- **和以前有什么不同**：相较许多单细胞基础模型偏重表征学习与下游判别，Lingshu-Cell显式建模“细胞状态分布”，能用于生成式仿真。相较依赖连续高斯假设或强基因筛选的方案，它采用离散扩散并直接处理大规模全基因token，更契合计数稀疏特征。进一步地，它把身份与扰动联合条件化，面向未见组合的反应预测，而不仅是已见条件重构。
- **你能学到什么**：阅读此文可重点关注三点：为何离散扩散比连续潜变量更适配scRNA计数与稀疏性；掩码训练如何帮助学习全局基因依赖。其次要看条件控制接口的设计，即身份变量与扰动变量如何共同注入生成过程。最后可思考评估维度从“单基因误差”转向“分布、亚群比例与生物学可解释性”的必要性。
- **关键洞见**：：要做虚拟细胞，核心不只是压缩表征，而是学习可条件采样的细胞状态世界模型。离散token化结合掩码扩散，使模型在全转录组层面同时保留稀疏性、组合依赖和可生成性。由此可在计算机中测试身份×扰动的反应空间，为机制探索与干预筛选提供更可扩展的试验场。

### 📡 Embodied Data Collection Devices（1）

#### ⭐ CARLA-Air: Fly Drones Inside a CARLA World -- A Unified Infrastructure for Air-Ground Embodied Intelligence
- **主题**：Embodied Data Collection Devices, 4DGS + Diffusion Simulator
- **重点命中**：embodied intelligence
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.28032.png
![CARLA-Air: Fly Drones Inside a CARLA World -- A Unified Infrastructure for Air-Ground Embodied Intelligence](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.28032.png)
- **这篇在做什么**：作者做了一个“二合一”仿真底座：在同一个 Unreal 进程里同时跑 CARLA 的城市交通和 AirSim 的多旋翼飞行动力学。这样无人机看到的车流、行人、红绿灯与地面智能体经历的是同一时钟和同一场景状态，不再需要桥接两个仿真器去对齐。结果是更适合做空地协作任务、跨模态数据采集与统一策略训练。
- **核心方法**：输入与表示层面，系统以共享城市场景、统一物理 tick、统一渲染管线作为世界状态表示，并在每个 tick 同步输出多平台多传感器观测。关键模块包括：CARLA 交通与行人生态、AirSim 飞行动力学栈、兼容 CARLA/AirSim Python API 与 ROS 2 的接口层、可扩展资产接入管线。流程上主要是单进程同步仿真与传感器采集，训练或推理侧可直接复用既有控制/学习代码，无需改写接口。
- **和以前有什么不同**：相较以往“驾驶仿真器只强地面、飞行仿真器只强空中”的分域方案，CARLA-Air把两者放在同一引擎进程内，避免桥接协同常见的时空不同步与通信开销。它的增量不只是功能叠加，更在于统一时钟下的物理一致性和跨智能体同步感知。另一个现实改进是继承并延续了已归档的 AirSim 飞行能力，降低生态断档风险。
- **你能学到什么**：读这篇可重点关注“单进程统一仿真”对数据质量的影响，尤其是多智能体交互任务里标签与观测的严格对齐。第二个学习点是接口兼容策略：保留原生 API 往往比重写新 API 更能促进社区迁移。实践上可把它当作空地协同基座，先跑现有 CARLA/AirSim 脚本，再逐步扩展到视觉语言行动或强化学习流水线。
- **关键洞见**：核心洞见是：空地具身智能的瓶颈不只在模型能力，更在仿真基础设施是否“同世界、同时钟、同物理”。只要底座做到跨平台严格同步，合作控制、联合感知和多模态数据构建都会显著顺畅。CARLA-Air体现了一个高性价比路径：最大化复用既有生态，同时最小化迁移摩擦。
