### 🧠 World Models（1）

#### ⭐ GameplayQA: A Benchmarking Framework for Decision-Dense POV-Synced Multi-Video Understanding of 3D Virtual Agents
- **主题**：World Models
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.24329.png
![GameplayQA: A Benchmarking Framework for Decision-Dense POV-Synced Multi-Video Understanding of 3D Virtual Agents](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.24329.png)
- **这篇在做什么**：可以把GameplayQA理解为一套“游戏版驾驶考试”：给模型看多人对战的第一人称同步视频，问它谁在做什么、何时发生、对局势有什么影响。数据标注非常密，持续记录自我、他人和环境三类信息，并据此构造不同难度问题。结果显示当前顶尖多模态模型离人类还有明显差距，尤其在时间对齐和角色归因上。
- **核心方法**：输入是多人3D游戏的POV同步视频，表示层采用Self/Other Agents/World三元结构，把状态、动作、事件按时间并发标注。关键模块包括高密度时序标注、由标注提炼出的2.4K诊断问答，以及配套的干扰项分类体系，用于定位幻觉来源。评测流程是按认知复杂度分级出题并统一打分，再结合干扰项统计分析模型在时序定位、跨视频对应和主体归因上的失误模式。
- **和以前有什么不同**：相较以往视频问答基准，GameplayQA强调“决策密度高、多人并发、第一人称同步”三点，而不只是离线叙事理解。它不仅考察是否答对，还通过结构化干扰项分析“为什么答错”，提升了可解释诊断能力。另一个增量是围绕智能体视角组织标注，更直接服务具身决策与世界建模研究。
- **你能学到什么**：读这篇可重点学习如何把任务分解为Self/Other/World，这种本体结构有助于统一标注规范与评测维度。其次可借鉴其“分级认知难度+干扰项 taxonomy”的设计，让基准具备诊断价值而非单一排行榜意义。最后要注意：高性能感知模型在高事件密度场景仍会暴露时序与归因短板，数据设计需主动放大这些能力差异。
- **关键洞见**：论文的核心洞见是：对智能体真正关键的不是静态语义识别，而是在高频变化中完成跨时刻、跨主体、跨视角的因果性对齐。只有把评测建立在同步多视频和并发事件上，模型的“看起来会描述”与“真的能支撑决策”差距才会被显性化。GameplayQA因此更像能力显微镜，而非普通题库。

### 🎮 4DGS + Diffusion Simulator（2）

#### ⭐ SpectralSplats: Robust Differentiable Tracking via Spectral Moment Supervision
- **主题**：4DGS + Diffusion Simulator
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.24036.png
![SpectralSplats: Robust Differentiable Tracking via Spectral Moment Supervision](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.24036.png)
- **这篇在做什么**：传统做法靠像素对齐来优化，但目标一旦偏到画面另一处，损失几乎不变，参数就学不动了。本文用整幅图的频谱特征来比较渲染图与目标图，即使两者完全不重叠，也能得到指向正确方向的梯度。再通过从低频到高频的逐步加难，让优化先“找得到目标”，再“贴得很精细”。
- **核心方法**：输入是当前参数化形变下由3DGS渲染的图像与目标帧图像，先将图像映射为一组全局复正弦响应（Spectral Moments）作为表示。关键模块包括频域监督损失与Frequency Annealing调度：前者提供全局吸引盆，后者按阶段提升频率成分以避免高频周期性局部极小。训练/推理时保持原可微渲染与参数更新流程不变，仅将空间光度损失替换或并联为频谱损失，实现对MLP或稀疏控制点等形变参数化的稳定优化。
- **和以前有什么不同**：相较以像素重合为前提的光度损失，这项方法的核心增量是构造了“无重叠也有梯度”的目标函数。相较直接上频域匹配的朴素方案，作者进一步给出有原则的频率退火，缓解高频引入的周期性陷阱。它不是改3DGS表示本身，而是作为可插拔监督替换，迁移到不同形变模型的成本较低。
- **你能学到什么**：阅读时可重点理解“局部支持高斯+空间损失=梯度稀疏/消失”的机制，这解释了许多失败案例并非优化器问题。其次把Spectral Moments看作全局锚点：低频负责粗定位，高频负责细对齐，与经典由粗到细配准思想一致。实现上可先做并联损失做消融，再尝试纯替换，以评估对现有管线的侵入性和收敛稳定性。
- **关键洞见**：最关键洞见是：跟踪失败并不总是因为模型表达力不足，而常是监督信号作用域太局部。把监督提升到全局频域后，优化景观从“到处平坦”变为“远处也有坡度”，从而显著扩大可收敛初始化范围。再用频率退火连接全局可达性与局部精度，兼顾鲁棒收敛与最终对齐质量。

#### ⭐ 4DGS360: 360° Gaussian Reconstruction of Dynamic Objects from a Single Video
- **主题**：4DGS + Diffusion Simulator
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.21618.png
![4DGS360: 360° Gaussian Reconstruction of Dynamic Objects from a Single Video](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.21618.png)
- **这篇在做什么**：作者想解决的是：只给一段普通手机视频，如何重建一个能从任意方向看、还会随时间变化的物体。以往方法常把“看到的表面”当成全部，遮挡区域会被错误补全，绕到背面就穿帮。4DGS360通过更可靠的3D点轨迹先把空间骨架搭好，再做高斯优化，因此360度观察更连贯。
- **核心方法**：输入是单目动态视频，先从2D跟踪中挑选高置信轨迹点作为锚点，并构造成3D表示；AnchorTAP3D在锚点约束下生成更稳健的3D点时序轨迹，抑制漂移并覆盖遮挡区。随后以这些3D轨迹初始化4D高斯表示，再进行联合优化，使几何、外观与时序运动在多帧中一致。推理时可在训练视角外的大角度相机位姿下渲染动态对象。
- **和以前有什么不同**：相较依赖2D先验或逐视角拟合的方案，4DGS360把“先有可信3D时序结构”放在优化前面，减少了只贴合可见面的偏置。其关键增量是AnchorTAP3D：利用2D高置信锚点强化3D轨迹，而非直接让2D信号主导几何。另一个贡献是iPhone360基准，专门考察与训练视角大夹角的360°泛化能力。
- **你能学到什么**：读这篇可重点关注“初始化决定上限”的观点：4DGS优化再强，坏初始化也会把遮挡区带偏。其次要理解锚点机制如何在长时序中抑制轨迹漂移，这是动态重建稳定性的关键。复现时建议先检查轨迹置信度筛选与遮挡段连续性，再调高斯优化参数。
- **关键洞见**：单目360°动态重建的瓶颈不只在渲染器，而在是否能获得对不可见区域友好的3D初始结构。把高置信2D观测转化为受约束的3D时序锚点，可显著缓解几何歧义并提升跨视角一致性。换言之，先稳住3D时空骨架，再让4DGS细化外观，才更可能实现可靠的全向重建。

### 📌 Other（1）

#### Can LLM Agents Be CFOs? A Benchmark for Resource Allocation in Dynamic Enterprise Environments
- **主题**：Other
- **论文链接**：
- **辅助图示**：https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.23638.png
![Can LLM Agents Be CFOs? A Benchmark for Resource Allocation in Dynamic Enterprise Environments](https://cdn-thumbnails.hf-mirror.com/social-thumbnails/papers/2603.23638.png)
- **这篇在做什么**：论文在一个长达132个月的企业模拟器里，让LLM代理扮演类似CFO的角色做预算与资源分配。代理无法直接看到全局状态，只能花预算调用组织工具获取信息，再据此做取舍。结果显示当前先进模型整体表现仍不稳，长期生存与持续优化都很困难。
- **核心方法**：输入与表示上，环境融合公司财务数据、匿名业务文档、宏观与行业信号及经营规则，形成部分可观测决策状态。关键模块是带预算的信息获取工具链与企业运营模拟器，代理需在“探查信息”和“保留资源”之间动态权衡。流程上按月迭代：先观测可得信号并决定是否花费预算查询，再执行资源配置动作，环境推进并反馈新状态，直至长周期结束或经营失败。
- **和以前有什么不同**：相较常见短时任务或静态问答评测，这项基准强调跨多年、连续承诺的资源配置与路径依赖。它不是单步最优选择，而是要求在不完备信息下兼顾当期目标和未来灵活性。论文还指出模型规模并不稳定对应更好结果，挑战了“更大即更强”的直觉。
- **你能学到什么**：阅读时可重点关注“部分可观测+预算化信息访问”这一设定，它逼近现实企业中调研成本与决策时滞。其次要区分短期收益与长期生存，两者在该任务中可能冲突。对后续研究，启发是把规划、风险控制和主动信息采样联动设计，而非只增强语言能力。
- **关键洞见**：LLM代理在长期企业资源配置上的瓶颈，不是单纯算力或参数规模不足，而是跨期权衡与不确定性管理能力不足。真正困难在于：何时花资源获取信息、何时保守、何时进攻，这些策略需随环境演化持续修正。该基准因此揭示了从“会推理”到“会经营”之间仍有显著鸿沟。
